# 订单余额逻辑修正

## 📋 问题总结

### 发现的问题

1. **消费订单创建时不扣除余额**
   - 测试下单时，用户余额没有被扣除
   - 这导致用户可以无限下单，不受余额限制

2. **退款逻辑不明确**
   - 订单失败是否应该退款？
   - 退款是否应该退还余额？

3. **余额变动逻辑不完整**
   - 缺少余额变动的日志记录

## ✅ 修正后的逻辑（最终版）

### 1. 消费订单（consume）

**创建订单时：**
- 检查用户余额是否足够
- 余额不足时拒绝创建订单
- 余额充足时立即扣除订单金额
- 记录余额变动日志

```javascript
// 消费订单创建流程
1. 检查余额 >= 订单金额
2. 余额 = 余额 - 订单金额
3. 记录余额变动日志（change_type: 'consume'）
4. 创建订单（状态：待处理）
```

**订单状态变更时：**
- **订单失败（status=3）**：不操作余额（不自动退款）
- **订单退款（status=4）**：退还余额（只有手动选择"退款"才退还）
- **订单成功（status=2）**：不操作余额（已经扣除过了）

```javascript
// 消费订单退款流程（仅退款时）
1. 余额 = 余额 + 订单金额
2. 记录余额变动日志（change_type: 'refund'）
3. 更新订单状态

// 消费订单失败流程（不退款）
// 只更新订单状态，不操作余额
```

### 2. 充值订单（recharge）

**创建订单时：**
- 不扣除余额（充值是用户往系统里充钱，还未确认到账）
- 创建订单（状态：待处理）

**订单状态变更时：**
- **订单成功（status=2）**：增加余额
  - 余额 = 余额 + 订单金额
  - 记录余额变动日志（change_type: 'recharge'）
- **订单失败（status=3）**：不操作余额（充值失败，不加余额）
- **订单退款（status=4）**：扣除余额（因为之前已经加过了）
  - 余额 = 余额 - 订单金额
  - 记录余额变动日志（change_type: 'refund'）

```javascript
// 充值订单成功流程
1. 余额 = 余额 + 订单金额
2. 记录余额变动日志（change_type: 'recharge'）
3. 更新订单状态为成功

// 充值订单失败流程
// 只更新订单状态，不操作余额

// 充值订单退款流程
1. 余额 = 余额 - 订单金额
2. 记录余额变动日志（change_type: 'refund'）
3. 更新订单状态为退款
```

## 📊 余额变动类型

在 `balance_logs` 表中，`change_type` 字段的含义：

| change_type | 含义 | 说明 |
|-------------|------|------|
| `consume` | 消费扣除 | 用户购买产品时扣除余额 |
| `refund` | 退款返还 | 订单失败或退款时返还余额 |
| `recharge` | 充值增加 | 用户充值时增加余额 |
| `manual_adjust` | 手动调整 | 管理员手动调整余额 |

## 🔄 完整流程示例

### 消费订单流程

**正常流程：**
```
1. 用户余额：1000元
2. 创建消费订单（话费10元）
   - 余额检查：1000 >= 10 ✓
   - 扣除余额：1000 - 10 = 990元
   - 记录日志：change_type='consume', amount=10
   - 订单状态：待处理

3. 订单处理成功
   - 订单状态：成功
   - 用户余额：990元（保持不变）
```

**失败流程（不退款）：**
```
1. 用户余额：1000元
2. 创建消费订单（话费10元）
   - 扣除余额：1000 - 10 = 990元
   - 订单状态：待处理

3. 订单处理失败
   - 订单状态：失败
   - 不操作余额：仍为990元（不自动退款）
   - 用户余额：990元
```

**退款流程（手动选择退款）：**
```
1. 用户余额：990元（已有失败订单）
2. 手动选择退款
   - 订单状态：退款
   - 退还余额：990 + 10 = 1000元
   - 记录日志：change_type='refund', amount=10
   - 用户余额：1000元
```

### 充值订单流程

**正常流程：**
```
1. 用户余额：1000元
2. 创建充值订单（充值100元）
   - 订单状态：待处理
   - 用户余额：1000元（不变）

3. 订单处理成功
   - 订单状态：成功
   - 增加余额：1000 + 100 = 1100元
   - 记录日志：change_type='recharge', amount=100
   - 用户余额：1100元
```

**退款流程：**
```
1. 用户余额：1100元（已有充值成功订单）
2. 申请退款
   - 订单状态：退款
   - 扣除余额：1100 - 100 = 1000元
   - 记录日志：change_type='refund', amount=100
   - 用户余额：1000元（恢复充值前状态）
```

## ⚠️ 重要注意事项

### 1. 防止重复退款

代码中通过检查 `oldStatus` 来防止重复退款：

```javascript
// 只有从非退款状态变为退款状态时才处理
if (status === 4 && oldStatus !== 4 && oldStatus === 2) {
  // 处理退款逻辑
}
```

### 2. 余额精度处理

所有余额计算都使用 `Math.round(balance * 100) / 100` 来保证两位小数精度：

```javascript
const balanceAfter = balanceBefore + parseFloat(order.amount);
db.get('members').find({ id: order.member_id })
  .assign({ balance: Math.round(balanceAfter * 100) / 100 }).write();
```

### 3. 余额检查

创建消费订单时必须先检查余额是否足够：

```javascript
if (balance < parseFloat(amount)) {
  return fail(res, '余额不足');
}
```

### 4. 余额变动日志

每次余额变动都会记录到 `balance_logs` 表，包括：
- `member_id`：用户ID
- `member_name`：用户名
- `change_type`：变动类型
- `amount`：变动金额
- `balance_before`：变动前余额
- `balance_after`：变动后余额
- `remark`：备注说明
