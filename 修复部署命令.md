# 🔧 修复部署命令

## 问题原因
`/tmp` 目录没有写入权限

## 解决方案

使用用户主目录或 `/app` 目录：

### 方法1：使用用户主目录（推荐）

```bash
cd ~ && curl -O https://raw.githubusercontent.com/pengx2301-art/cydzhanlue/main/deploy-remote.sh && chmod +x deploy-remote.sh && ./deploy-remote.sh
```

### 方法2：使用 /app 目录

```bash
cd /app && curl -O https://raw.githubusercontent.com/pengx2301-art/cydzhanlue/main/deploy-remote.sh && chmod +x deploy-remote.sh && ./deploy-remote.sh
```

### 方法3：分步执行（更清晰）

```bash
# 进入主目录
cd ~

# 下载脚本
curl -O https://raw.githubusercontent.com/pengx2301-art/cydzhanlue/main/deploy-remote.sh

# 赋予执行权限
chmod +x deploy-remote.sh

# 执行脚本
./deploy-remote.sh
```

---

## 如果curl还是失败

### 使用wget替代：

```bash
cd ~ && wget https://raw.githubusercontent.com/pengx2301-art/cydzhanlue/main/deploy-remote.sh && chmod +x deploy-remote.sh && ./deploy-remote.sh
```

### 或者手动创建：

```bash
cd ~
nano deploy-remote.sh
```

然后复制粘贴脚本内容，保存后执行：

```bash
chmod +x deploy-remote.sh
./deploy-remote.sh
```
