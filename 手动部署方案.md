# 🔧 手动部署方案（绕过GitHub连接问题）

由于服务器连接GitHub有网络问题，我们使用手动部署方式。

---

## 📦 方案：本地打包上传到服务器

### 步骤1：在本地创建部署包

我已经为您准备好了所有文件，现在需要打包上传到服务器。

**在Windows上执行（PowerShell）：**

```powershell
cd c:\Users\60496\WorkBuddy\Claw

# 创建部署目录
New-Item -ItemType Directory -Force -Path "c:\temp\cyd-deploy"

# 复制文件
Copy-Item "api-server.js" "c:\temp\cyd-deploy\"
Copy-Item "product-connection.js" "c:\temp\cyd-deploy\"
Copy-Item "order-module.js" "c:\temp\cyd-deploy\"
Copy-Item "package.json" "c:\temp\cyd-deploy\"
Copy-Item "product-connection.html" "c:\temp\cyd-deploy\"
Copy-Item "order-page.html" "c:\temp\cyd-deploy\"

# 创建压缩包
Compress-Archive -Path "c:\temp\cyd-deploy\*" -DestinationPath "c:\temp\cyd-v2.zip" -Force

Write-Host "部署包已创建: c:\temp\cyd-v2.zip" -ForegroundColor Green
```

### 步骤2：上传到服务器

**方法A：使用SCP（如果有Git Bash）**

打开Git Bash，执行：

```bash
scp /c/temp/cyd-v2.zip ubuntu@81.70.208.147:/app/
```

输入密码：`Aa462300`

**方法B：使用FTP/SFTP工具**

使用FileZilla、WinSCP等工具上传 `c:\temp\cyd-v2.zip` 到服务器的 `/app/` 目录

### 步骤3：在服务器上解压并部署

连接服务器后执行：

```bash
ssh ubuntu@81.70.208.147
# 密码: Aa462300

cd /app

# 解压文件
unzip -o cyd-v2.zip

# 删除压缩包
rm cyd-v2.zip

# 安装依赖
npm install

# 重启服务
pm2 restart cyd-admin || pm2 start api-server.js --name cyd-admin

# 保存配置
pm2 save

# 查看状态
pm2 status
```

## 🚀 更简单的方法：直接复制粘贴

如果上传文件麻烦，可以直接在服务器上创建文件：

### 方法：使用cat命令创建文件

```bash
ssh ubuntu@81.70.208.147
# 密码: Aa462300

cd /app

# 创建api-server.js
cat > api-server.js << 'EOF'
[粘贴api-server.js的内容]
EOF

cat > product-connection.js << 'EOF'
[粘贴product-connection.js的内容]
EOF

# 创建order-module.js
cat > order-module.js << 'EOF'
[粘贴order-module.js的内容]
EOF

# 创建package.json
cat > package.json << 'EOF'
[粘贴package.json的内容]
EOF

# 创建HTML文件
cat > product-connection.html << 'EOF'
[粘贴product-connection.html的内容]

[粘贴order-page.html的内容]
EOF

# 安装依赖并启动
npm install
pm2 restart cyd-admin || pm2 start api-server.js --name cyd-admin
pm2 save
```

## 📋 最简化的部署命令

我已经为您创建了简化版部署脚本，只需要在服务器上执行：

```bash
# 进入应用目录
cd /app

# 备份旧文件（可选）
mkdir -p backup && cp *.js *.json *.html backup/ 2>/dev/null || true

# 安装依赖（如果还没有）
npm install

# 重启服务
pm2 restart cyd-admin

# 查看状态
pm2 status
```

**前提是您已经把文件上传到 /app 目录了**
