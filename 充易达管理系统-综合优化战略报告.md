## 数据库架构优化

### P0 级别严重问题

#### 1. 缺失核心订单表
**问题描述**: 充值订单完全无法持久化,导致:
- 所有交易记录丢失
- 财务统计全部为随机假数据
- 无法查询历史订单
- 无法对账和审计

**解决方案**: 创建完整的订单管理体系

#### 2. 双代理表结构混乱
**问题描述**: `members` 和 `agents` 两个表同时存在,导致:
- 余额更新逻辑错误
- 数据同步困难
- 查询效率低下

**解决方案**: 统一为 `members` 表,通过角色字段区分

#### 3. 财务数据全为假数据
**问题描述**: `daily_revenue` 使用随机生成:
```javascript
"revenue": Math.floor(Math.random() * 10000)
```

**解决方案**: 基于真实订单数据计算

### P1 级别重要问题

#### 1. 时间格式不统一
**问题**: 混用 `2026/3/16` 和 `2026-03-16` 格式

**影响**:
- SQL查询失败
- 时间排序错误
- 跨时区问题

**解决方案**: 统一使用 ISO 8601 格式 (`YYYY-MM-DDTHH:mm:ss.sssZ`)

#### 2. ID生成机制风险
**问题**: `nextId()` 百万级数据时可能调用栈溢出

**解决方案**: 实现批量ID生成或改用UUID

### 核心函数设计 (8个)

#### 1. 创建订单
```sql
DELIMITER $$
CREATE PROCEDURE sp_create_order(
  IN p_member_id VARCHAR(36),
  IN p_product_id VARCHAR(36),
  IN p_recharge_account VARCHAR(100),
  IN p_quantity INT,
  OUT p_order_id VARCHAR(36),
  OUT p_status INT
)
BEGIN
  DECLARE v_product_price DECIMAL(10,2);
  DECLARE v_product_cost DECIMAL(10,2);
  DECLARE v_member_balance DECIMAL(10,2);
  DECLARE v_supplier_id VARCHAR(36);
  DECLARE v_order_no VARCHAR(32);
  DECLARE v_total_price DECIMAL(10,2);
  DECLARE v_total_cost DECIMAL(10,2);
  DECLARE v_gross_profit DECIMAL(10,2);

  -- 获取产品信息
  SELECT price, cost_price, supplier_id
  INTO v_product_price, v_product_cost, v_supplier_id
  FROM products WHERE id = p_product_id;

  -- 生成订单号
  SET v_order_no = CONCAT('ORD', DATE_FORMAT(NOW(), '%Y%m%d'), LPAD(FLOOR(RAND() * 1000000), 6, '0'));

  -- 计算总价
  SET v_total_price = v_product_price * p_quantity;
  SET v_total_cost = v_product_cost * p_quantity;
  SET v_gross_profit = v_total_price - v_total_cost;

  -- 插入订单
  INSERT INTO orders (
    id, order_no, member_id, product_id, supplier_id,
    recharge_account, amount, unit_price, cost_price,
    gross_profit, quantity, total_price, total_cost, status
  ) VALUES (
    UUID(), v_order_no, p_member_id, p_product_id, v_supplier_id,
    p_recharge_account, v_product_price, v_product_price, v_product_cost,
    v_gross_profit, p_quantity, v_total_price, v_total_cost, 'pending'
  );

  SET p_order_id = LAST_INSERT_ID();
  SET p_status = 0; -- 成功
END$$
DELIMITER ;
```

#### 3. 扣除余额
```sql
DELIMITER $$
CREATE PROCEDURE sp_deduct_balance(
  IN p_member_id VARCHAR(36),
  IN p_order_id VARCHAR(36),
  IN p_amount DECIMAL(10,2),
  OUT p_result INT
)
BEGIN
  DECLARE v_current_balance DECIMAL(10,2);
  DECLARE v_transaction_id VARCHAR(36);

  -- 检查余额
  SELECT balance INTO v_current_balance FROM members WHERE id = p_member_id;

  IF v_current_balance < p_amount THEN
    SET p_result = -1; -- 余额不足
  ELSE
    -- 扣除余额
    UPDATE members SET balance = balance - p_amount WHERE id = p_member_id;

    -- 记录交易
    SET v_transaction_id = UUID();
    INSERT INTO transactions (
      id, member_id, type, amount, balance_after, order_id, description, created_at
    ) VALUES (
      v_transaction_id, p_member_id, 'order', -p_amount,
      v_current_balance - p_amount, p_order_id,
      CONCAT('订单消费: ', p_order_id), NOW()
    );

    SET p_result = 0; -- 成功
  END IF;
END$$
DELIMITER ;
```

#### 4. 计算财务统计
```sql
DELIMITER $$
CREATE PROCEDURE sp_calculate_daily_statistics(
  IN p_date DATE,
  OUT p_total_revenue DECIMAL(15,2),
  OUT p_total_profit DECIMAL(15,2),
  OUT p_success_orders INT,
  OUT p_failed_orders INT
)
BEGIN
  SELECT
    COALESCE(SUM(total_price), 0),
    COALESCE(SUM(gross_profit), 0),
    SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END),
    SUM(CASE WHEN status = 'failed' THEN 1 ELSE 0 END)
  INTO p_total_revenue, p_total_profit, p_success_orders, p_failed_orders
  FROM orders
  WHERE DATE(created_at) = p_date;
END$$
DELIMITER ;
```

#### 6. 更新供应商指标
```sql
DELIMITER $$
CREATE PROCEDURE sp_update_supplier_metrics(
  IN p_supplier_id VARCHAR(36),
  IN p_period VARCHAR(7)
)
BEGIN
  INSERT INTO supplier_metrics (
    supplier_id, period, total_orders, success_orders, failed_orders,
    success_rate, total_amount, total_cost, total_profit
  )
  SELECT
    p_supplier_id,
    p_period,
    COUNT(*) AS total_orders,
    SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) AS success_orders,
    SUM(CASE WHEN status = 'failed' THEN 1 ELSE 0 END) AS failed_orders,
    ROUND(SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS success_rate,
    COALESCE(SUM(total_price), 0) AS total_amount,
    COALESCE(SUM(total_cost), 0) AS total_cost,
    COALESCE(SUM(gross_profit), 0) AS total_profit
  FROM orders
  WHERE supplier_id = p_supplier_id
    AND DATE_FORMAT(created_at, '%Y-%m') = p_period
  ON DUPLICATE KEY UPDATE
    total_orders = VALUES(total_orders),
    success_orders = VALUES(success_orders),
    failed_orders = VALUES(failed_orders),
    success_rate = VALUES(success_rate),
    total_amount = VALUES(total_amount),
    total_cost = VALUES(total_cost),
    total_profit = VALUES(total_profit);
END$$
DELIMITER ;
```

```sql
DELIMITER $$
CREATE PROCEDURE sp_create_referral_code(
  IN p_inviter_id VARCHAR(36),
  IN p_name VARCHAR(100),
  IN p_description TEXT,
  IN p_reward_type VARCHAR(20),
  IN p_reward_value DECIMAL(10,2),
  IN p_max_uses INT,
  IN p_expires_at DATETIME,
  OUT p_code VARCHAR(20)
)
BEGIN
  DECLARE v_code VARCHAR(20);
  DECLARE v_exists INT;

  -- 生成唯一推荐码
  WHILE TRUE DO
    SET v_code = CONCAT('CYD', LPAD(FLOOR(RAND() * 1000000), 6, '0'));
    SELECT COUNT(*) INTO v_exists FROM referral_codes WHERE code = v_code;
    IF v_exists = 0 THEN
      LEAVE;
    END IF;
  END WHILE;

  -- 插入推荐码
  INSERT INTO referral_codes (
    id, code, inviter_id, name, description,
    reward_type, reward_value, max_uses, is_active, expires_at
  ) VALUES (
    UUID(), v_code, p_inviter_id, p_name, p_description,
    p_reward_type, p_reward_value, p_max_uses, TRUE, p_expires_at
  );

  SET p_code = v_code;
END$$
DELIMITER ;
```

#### 8. 获取级别定价
```sql
DELIMITER $$
CREATE PROCEDURE sp_get_level_price(
  IN p_product_id VARCHAR(36),
  IN p_member_level VARCHAR(20),
  OUT p_price DECIMAL(10,2)
)
BEGIN
  DECLARE v_product_price DECIMAL(10,2);
  DECLARE v_level_price DECIMAL(10,2);

  -- 获取产品默认价格
  SELECT price INTO v_product_price FROM products WHERE id = p_product_id;

  -- 获取级别价格
  SELECT price INTO v_level_price
  FROM level_pricing
  WHERE product_id = p_product_id AND member_level = p_member_level
  LIMIT 1;

  -- 优先使用级别价,否则使用产品价
  IF v_level_price IS NOT NULL THEN
    SET p_price = v_level_price;
  ELSE
    SET p_price = v_product_price;
  END IF;
END$$
DELIMITER ;
```

### 三层定价机制

```
个人价 > 级别价 > 产品默认价

使用优先级:
1. 优先使用成员级别定价 (level_pricing)
2. 如果无级别定价,使用产品默认定价 (products.price)
3. 个人定价可在订单级别覆盖 (预留扩展)
```

### 3. 利润计算器

**文件**: `profit-calculator.html`

**功能**:
- 实时计算充值利润
- 可视化利润占比
- 历史计算记录
- 导出计算结果

**计算逻辑**:
```
利润 = (售价 - 成本) × 数量
利润率 = 利润 / 售价 × 100%
```

**界面元素**:
- 产品选择下拉框

- 售价/成本显示
- 利润实时计算卡片

**交互优化**:
- 输入框防抖,延迟300ms计算
- 利润为负数时红色高亮
- 添加到收藏夹功能

### orders 表 v2.0 字段设计

#### 基础字段
```sql
id              VARCHAR(36) PRIMARY KEY
order_no        VARCHAR(32) UNIQUE NOT NULL        -- 订单编号
member_id       VARCHAR(36) NOT NULL               -- 下单代理ID
product_id      VARCHAR(36) NOT NULL               -- 产品ID
supplier_id     VARCHAR(36) NOT NULL               -- 供应商ID
```

#### 金额字段
```sql
amount          DECIMAL(10,2) NOT NULL            -- 充值金额
unit_price      DECIMAL(10,2) NOT NULL            -- 单价
cost_price      DECIMAL(10,2) NOT NULL            -- 成本价
gross_profit    DECIMAL(10,2)                    -- 毛利润
quantity        INT DEFAULT 1                    -- 数量
total_price     DECIMAL(10,2) NOT NULL           -- 总价
total_cost      DECIMAL(10,2)                    -- 总成本
```

#### 状态字段
```sql
status          VARCHAR(20) NOT NULL             -- 订单状态
trade_no        VARCHAR(64)                      -- 第三方交易单号
error_message   TEXT                             -- 错误信息
```

#### 时间字段
```sql
created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
completed_at    DATETIME                         -- 完成时间
```

#### 索引设计
```sql
INDEX idx_member_id (member_id)
INDEX idx_product_id (product_id)
INDEX idx_supplier_id (supplier_id)
INDEX idx_status (status)
INDEX idx_created_at (created_at)
INDEX idx_order_no (order_no)
```

## 风险与建议

### 风险识别

#### 1. 数据迁移风险 ⚠️ 高风险
**风险描述**: 从 LowDB 迁移到 CloudBase 可能导致数据丢失或损坏

**应对措施**:
- 提前备份数据
- 开发数据校验脚本
- 分批次迁移
- 保留原始数据至少3个月

---

#### 2. 性能风险 ⚠️ 中风险
**风险描述**: 订单量增长后可能出现性能瓶颈

**应对措施**:
- 建立索引优化
- 实施读写分离
- 引入缓存机制
- 定期性能监控

---

#### 3. 安全风险 ⚠️ 高风险
**风险描述**: 财务数据安全,防止数据泄露

**应对措施**:
- 敏感数据加密
- 实施审计日志
- 定期安全扫描
- 权限分级管理

---

#### 4. 用户接受度风险 ⚠️ 中风险
**风险描述**: 新功能用户可能不适应

**应对措施**:
- 充分的用户培训
- 保留旧版本入口
- 收集用户反馈
- 快速迭代优化
