## 后端API说明

### GET /api/orders
**查询参数**：
- `page`: 页码（默认1）
- `limit`: 每页数量（默认20）
- `keyword`: 搜索关键词（订单号、用户名、产品名）
- `status`: 状态筛选（0=待处理、1=处理中、2=成功、3=失败、4=已退款）
- `type`: 类型筛选（consume=消费、recharge=充值）

**返回数据**：
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "total": 10,        // 总数
    "pending": 2,       // 待处理数量
    "success": 7,       // 成功数量
    "failed": 1,        // 失败数量
    "page": 1,
    "limit": 20,
    "items": [...]      // 订单列表
  }
}
```

## 使用说明

### 查看订单
1. 点击左侧菜单"订单管理"
2. 页面自动加载所有订单
3. 显示统计数据：全部、成功、处理中、失败

### 筛选订单
1. **按状态筛选**：点击状态统计条上的"全部"、"成功"、"处理中"、"失败"按钮
2. **按类型筛选**：在"全部类型"下拉框选择"消费"或"充值"
3. **按日期筛选**：选择开始日期和结束日期（当前未实现后端日期筛选）
4. **关键词搜索**：在输入框输入订单号、用户名或产品名，按回车或点击"搜索"按钮

### 重置筛选
点击"重置"按钮，清空所有筛选条件，显示所有订单

## 技术要点

1. **状态值对应关系**：
   - 前端下拉框：value为字符串 "0", "1", "2", "3", "4"
   - 后端API：status为数字 0, 1, 2, 3, 4
   - 数据库：status为数字 0, 1, 2, 3, 4

2. **金额显示**：
   - 使用 `Number(o.amount || 0).toFixed(2)` 保留两位小数
   - 使用 `num` CSS类实现右对齐

3. **安全防护**：
   - 所有用户输入都使用 `escHtml()` 函数进行HTML转义，防止XSS攻击

4. **无数据处理**：
   - 当订单列表为空时，显示居中的"暂无数据"提示
   - colspan="10" 跨越所有列
