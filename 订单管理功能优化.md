### 1. 修复分页显示错误

**问题：**
- 显示232页，但实际只有3条记录

- 更新底部统计显示格式：`共 X 条记录，当前第 Y/Z 页`

**修复位置：**

```javascript
// dashboard-ui/charts.js 第4078-4082行
const totalPages = Math.ceil((data.total || 0) / limit);
totalEl.textContent = `共 ${data.total || 0} 条记录，当前第 ${page}/${totalPages || 1} 页`;
```

### 2. 优化操作按钮

**优化前：**
- 只有一个"详情"按钮
- 需要点击"详情"打开模态框才能操作订单
- 操作流程繁琐

**优化后：**
- 在列表中直接显示操作按钮
- 根据订单状态动态显示不同的操作按钮
- 保留"详情"按钮用于查看详细信息

**不同状态的操作按钮：**

| 订单状态 | 显示的按钮 |
|---------|----------|
| 待处理（0） | 处理中、成功、失败 |
| 处理中（1） | 成功、失败 |
| 成功（2） | 退款 |
| 失败（3） | 重试、退款 |
| 已退款（4） | 已退款（文字） |

**实现代码：**
```javascript
// 根据订单状态生成操作按钮
switch (o.status) {
  case 0: // 待处理
    actionButtons = `
      <button class="btn btn-sm btn-warning" onclick="orderQuickUpdate(${o.id}, 1)">处理中</button>
      <button class="btn btn-sm btn-success" onclick="orderQuickUpdate(${o.id}, 2)">成功</button>
      <button class="btn btn-sm btn-danger" onclick="orderQuickUpdate(${o.id}, 3)">失败</button>
    `;
    break;
  // ... 其他状态
}
```

### 3. 添加批量操作功能

**功能描述：**
- 选中多个订单后，顶部显示批量操作按钮
- 可以批量更新订单状态
- 逐个更新订单，并显示成功/失败统计

**批量操作按钮：**
- 标记为处理中
- 标记为成功
- 标记为失败
- 退款

**使用方法：**
1. 勾选需要操作的订单复选框
2. 顶部自动显示批量操作区域
3. 点击相应的状态按钮
4. 确认批量操作
5. 查看操作结果

**实现代码：**
```javascript
// 批量更新订单状态
async function orderBatchUpdate(newStatus) {
  const orderIds = Array.from(document.querySelectorAll('.order-checkbox:checked'))
    .map(cb => parseInt(cb.getAttribute('data-id')));
  
  // 逐个更新
  for (const orderId of orderIds) {
    await apiRequest('PUT', `/api/orders/${orderId}/status`, {
      status: newStatus,
      auto_refund: false
    });
  }
}
```

**智能显示：**
- 未选中订单时，批量操作区域隐藏
- 选中订单后，批量操作区域自动显示

### 4. 新增函数

#### `orderQuickUpdate(orderId, newStatus)`
快速更新单个订单状态

**参数：**
- `orderId`: 订单ID
- `newStatus`: 新状态（0=待处理, 1=处理中, 2=成功, 3=失败, 4=已退款）

**功能：**
- 弹出确认对话框
- 调用API更新订单状态
- 显示成功提示
- 自动刷新订单列表

#### `orderBatchUpdate(newStatus)`
批量更新订单状态

**参数：**
- `newStatus`: 新状态

**功能：**
- 获取所有选中的订单ID
- 弹出确认对话框
- 逐个更新订单状态
- 显示操作结果统计
- 清空选择并刷新列表

#### `ordersAddBatchActions()`
添加批量操作UI

**功能：**
- 在搜索筛选区域添加批量操作按钮
- 监听复选框变化
- 智能显示/隐藏批量操作区域

## 📝 注意事项

1. **批量操作是逐个更新的**
   - 不是一次性更新多个订单
   - 每个订单都会单独调用API
   - 如果某个订单更新失败，不会影响其他订单

2. **批量操作结果统计**
   - 显示成功和失败的订单数量
   - 可以了解哪些订单更新失败

3. **管理员手动操作**
   - 所有快速操作和批量操作都设置 `auto_refund: false`
   - 失败状态不会自动退款
   - 需要手动点击"退款"按钮才退款

4. **复选框选择**
   - 选择后可以快速批量操作
   - 刷新列表后会清空选择
   - 可以随时取消选择
