### 步骤2: 配置SSH密钥（如果还没有）

## 🔧 常见问题

### 问题1: SSH连接超时
**解决方案**:
```bash
# 检查网络连接
ping 81.70.208.147

# 检查SSH端口
telnet 81.70.208.147 22

# 检查防火墙
ssh root@81.70.208.147 "iptables -L -n"
```

### 问题2: PM2命令不存在
**解决方案**:
```bash
# 安装PM2
npm install -g pm2

# 或使用npx
npx pm2 start api-server.js --name api-server --watch
```

### 问题3: 端口8899已被占用
**解决方案**:
```bash
# 查找占用进程
lsof -i :8899
# 或
netstat -tlnp | grep 8899

# 杀死进程
kill -9 <PID>
```

### 问题4: 文件权限问题
**解决方案**:
```bash
# 修改文件权限
chmod +x /root/Claw/*.js
chmod 644 /root/Claw/*.html

# 修改目录权限
chmod 755 /root/Claw/user
chmod 755 /root/Claw/admin
```

### 问题5: API服务器启动失败
**解决方案**:
```bash
# 查看详细日志
pm2 logs api-server --lines 100

# 检查Node.js版本
node --version
# 建议版本: v14.x 或更高

# 检查依赖
npm install
```

## 🔄 回滚方案

如果部署出现问题，可以快速回滚：

### 方法1: 使用备份
```bash
# 登录到服务器
ssh root@81.70.208.147

# 恢复备份文件
cd /root/Claw
cp backup/api-server.js api-server.js
cp backup/admin.js admin.js
cp backup/supplier-manager.js supplier-manager.js

# 重启服务器
pm2 restart api-server
```

### 方法2: 使用Git回退
```bash
# 登录到服务器
ssh root@81.70.208.147

# 查看提交历史
cd /root/Claw
git log --oneline -10

# 回退到上一个版本
git reset --hard HEAD~1

# 重启服务器
pm2 restart api-server
```

## 📈 监控和维护

### 日常监控
```bash

pm2 status

pm2 logs api-server

# 查看资源使用
```

### 定期备份
```bash
# 创建备份脚本
cat > /root/Claw/backup.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p /root/Claw/backup/$DATE
cp -r /root/Claw/*.js /root/Claw/backup/$DATE/
cp -r /root/Claw/*.html /root/Claw/backup/$DATE/
echo "Backup created: $DATE"
EOF

chmod +x /root/Claw/backup.sh

# 每天自动备份
crontab -e
# 添加: 0 2 * * * /root/Claw/backup.sh
```
